Know what is being logged. Avoid NPEs log.debug(Processing request with id: {}, request.getId()); This can cause an NPE.
Be concise and descriptive while logging messages Each logging statement should contain both data and description. log.debug(Message with id '{}' processed, message.getJMSMessageID());This uses SLf4J logging framework, by far the best framework
Log exceptions properly log.error(Error reading config file, e); Don't include exception message, as it will be printed automatically after the log statement, preceding stack trace. But in order to do so, you must pass the exception itself as the second argument.
