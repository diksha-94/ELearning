Workflow

Determines the tasks and actors involved in the Item creation and maintenance process. In addition, the execution of Item Creation rules, validations, approvals & rejections and other functionalities like managing item conflicts, reuse of existing items etc. are determined in the EMI Workflow

The main features of the EMI Workflow are:

  • Clarity of Ownership
    • Who does what
    • Shows under whose bucket the current task is residing

  • Event Management
    • Creation of items based on the event it caters to. Eg: Whether the item is being created for an 'Annual Event' like Christmas

  • Supplier visibility of the Item Creation/Maintenance status
    • Supplier can view if there are any errors that have to be cleared
    • The approval status of the items created
    • Existing conflicts with the submitted items
